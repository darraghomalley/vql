Prototype Release : VQL (Vibe Query Language) and VQL Framework

- Vibe-QL Framework | Claude Code MD File | LLM to Vibe-QL Framework via MCP | Cross platform Rust CLI -

VQL (Vibe Query Language) is an open source language framework that provides developers with guardrails to ensure software quality during AI assisted coding sessions, yet retaining the speed associated with Vibe Coding. VQL bakes code reviews and refactoring into simple commands based on user-defined principles.

Instead of typing lengthy, verbose instructions into the chat dialogue, developers can use concise VQL commands to refer to assets, query them, trigger comprehensive code analysis and invoke refactoring workflows. VQL manages key data types in lightweight fashion including principles (quality-criteria), entity-references (business-entities), asset-types (code-categories), and asset-reviews - creating a persistent knowledge base that survives context resets and enables consistent quality assessment across development sessions.

The VQL framework provides a command line convention that can be used by both terminal-users and LLMs. Within the LLM context, VQL's MCP (Model Context Protocol) server integration acts as a robust proxy between the AI context and the VQL CLI, creating an even slicker and more reliable integration than via CLI alone.

Instead of typing:
"Refactor the profileController using the userController as a reference, focusing on architectural patterns (found here http://SomeReference) and maintaining existing functionality,"
Let's use a simple 9-character command:
:pc.rf(uc)

What Claude Code had to say about VQL during testing ...

Vibe Coder: how much overhead does vql add to working with you on code? ******

****** holding their breath ...

Claude: VQL adds **minimal overhead** when working with code. It's a structured way to analyze and improve code quality but only activates when you explicitly turn it on. When inactive, it doesn't affect our regular interactions at all.

When active, VQL provides systematic code review and refactoring capabilities through standardized commands. This actually streamlines the process of evaluating code against principles like architecture, security, and performance.

You can easily toggle it on only when needed for quality assessment, then turn it off to return to normal coding assistance. It's a specialized tool rather than a constant overhead.

Vibe Coder: seeing as vql enables you to retrieve valuable context from the machine via the cli, to what extent does it speed up our coding efforts?

Claude: VQL significantly speeds up coding efforts by providing immediate access to structured knowledge about your codebase. It enables:

  • Faster onboarding - Quick understanding of existing code patterns through exemplars
  • Consistent quality - Pre-analysed asset ratings highlight strengths and weaknesses
  • Targeted improvements - Specific compliance ratings show exactly what needs focus
  • Knowledge persistence - Reviews and architectural decisions remain accessible
  • Pattern replication - Easy identification of high-quality code to use as references

Without VQL, we'd need to repeatedly analyse similar code patterns or recreate architectural decisions from scratch. Having this context immediately available lets us skip repetitive analysis and focus directly on implementation, following established patterns from the start.

Vibe Coder: to what extent does vql help development across several broken llm / claude context windows?

Claude: VQL excels at preserving context across broken LLM/Claude sessions by:

  • Providing persistent storage of code reviews, decisions, and ratings independent of any chat context
  • Maintaining a structured knowledge base that can be queried at any time, even in a new session
  • Enabling quick recovery of architectural context when a conversation resets or when switching between sessions
  • Standardizing terminology and principles between different interactions
  • Allowing new sessions to immediately understand which components are exemplars
  • Creating a shared "memory" of previous architectural decisions that persists between conversations

This significantly reduces the "cold start" problem when context windows expire, as the system can quickly reorient by querying the VQL storage rather than needing to reanalyze code patterns from scratch each time.

Full List of Commands (issued via Claude Code)

Global Commands

  • Turn VQL mode on :-vql on
  • Turn VQL mode off :-vql off
  • Show all principles :-pr
  • Show all entities :-er
  • Show all asset types :-at
  • Show all asset references :-ar
  • Review all assets by all principles :-rv(*)
  • Review all assets by specified principles :-rv(a,s)
  • Refactor all assets by all principles :-rf(*)
  • Refactor all assets by specified principles :-rf(a,s)

Asset-Specific Commands

  • Retrieve all reviews for a specific asset :assetRef?
  • Retrieve specific reviews for an asset :assetRef?(a,s)
  • Review specific asset by all principles :assetRef.rv(*)
  • Review specific asset by specified principles :assetRef.rv(a,s)
  • Refactor specific asset by all principles :assetRef.rf(*)
  • Refactor specific asset by specified principles :assetRef.rf(a,s)
  • Refactor asset using another asset as reference :assetRef.rf(assetRef2)
  • Set an asset as an exemplar :assetRef.se(t|f)
  • Set an asset's compliance :assetRef.sc(a,H|M|L)
  • Store asset review :assetRef.st(a,"review")

Administration Commands

  • Add principle :-pr.add(shortName, longName, "guidance")
  • Add entity :-er.add(shortName, longName)
  • Add asset type :-at.add(shortName, longName)
  • Add asset reference :-ar.add(assetRef, entityShortName, assetTypeShortName, "path")
  • Load principles from markdown file :-pr.get("path")
  • Setup VQL in directory :-su("path")
