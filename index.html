New: Enhanced refactoring with reference assets and clearer -pr syntax

✨ Latest Updates

  • Clearer Syntax: -pr = all principles (replaces *)
  • Reference Refactoring: :pc.rf(-pr, uc) or :tm.rf(a,s,uc,pc)
  • Unified Namespace: Unique names enable mixing principles & assets: :tm.rf(a,s,uc,pc)
  • Data Integrity: Referential integrity + smart cascading for safe operations
  • Lifecycle Management: Rename with :rn(), delete with cascade rules :dl()
  • Enhanced Workflows: Post-refactor reviews mandatory, mixed principle lists supported

VQL (Vibe Query Language) is an open source framework providing quality guardrails for AI-assisted coding. It transforms verbose instructions into concise commands, managing principles (quality criteria), entities (business objects), asset types (code categories), and reviews in a persistent knowledge base that survives context resets.

Works seamlessly via CLI or MCP server integration. Built with safety in mind through unified namespace (unique names across all types) and referential integrity (dependencies must exist before use).

Supports complete lifecycle management with :rn() for renaming and :dl() for deletion with smart cascading—automatically cleaning up dependent assets while protecting your review history.

Instead of: "Refactor profileController using all principles with userController as reference..."
Just type: :pc.rf(-pr, uc)

Initialize VQL Environment
Command:
:-su("C:/App")
Action:
Creates VQL directory and initializes storage
CLI Command:
vql -su "C:/App"
Load Principles from External File
Command:
:-pr.get("C:/principles.md")
Action:

CLI Command:
vql -pr -get "C:/principles.md"

Review Asset Against Principle
Command:
:uc.rv(a)
Action:
Reviews UserController against architecture principle and stores result
CLI Command:
vql -st uc a "The UserController demonstrates excellent separation of concerns with clean dependency injection and proper error handling. HIGH compliance"
Mark Asset as Exemplar
Command:
:uc.se(t)
Action:
Marks UserController as exemplar reference implementation
CLI Command:
vql -se uc t
Refactor Asset with Reference
Command:
:pc.rf(-pr, uc)
Action:
Refactors ProfileController using all principles with UserController as reference, then mandates review
CLI Command:
vql -st pc a "The ProfileController now demonstrates excellent separation of concerns with clean dependency injection and proper error handling. HIGH compliance"
Review Asset Against Multiple Principles
Command:
:tm.rv(a,s)
Action:
Reviews TeamModel against architecture and security principles (unified namespace enables mixing)
CLI Command:
vql -st tm a "Clean domain model with proper encapsulation. HIGH" then vql -st tm s "Implements secure data validation. HIGH"
Refactor with Mixed Parameters
Command:
:tm.rf(a,s,uc,pc)
Action:
Refactors TeamModel for architecture & security using both UserController and ProfileController as references
CLI Command:
vql -rf tm a,s uc,pc (unified namespace distinguishes principles from assets)

Complete Command Reference

Global Commands

  • VQL on/off :-vql on | :-vql off
  • Show principles :-pr
  • Show entities :-er
  • Show asset types :-at
  • Show assets :-ar
  • Review all assets :-rv(-pr) | :-rv(a,s)
  • Refactor all assets :-rf(-pr) | :-rf(a,s)

Asset-Specific Commands

  • Get reviews :assetRef?
  • Get specific reviews :assetRef?(a,s)
  • Review asset :assetRef.rv(-pr) | :assetRef.rv(a,s)
  • Refactor asset :assetRef.rf(-pr) | :assetRef.rf(a,s)
  • Refactor with reference :assetRef.rf(-pr,pc) or :assetRef.rf(a,s,pc,tm)
  • Set exemplar :assetRef.se(t|f)
  • Set compliance :assetRef.sc(a,H|M|L)
  • Store review :assetRef.st(a,"review")

Administration Commands

  • Add principle :-pr.add(shortName, longName, "guidance")
  • Add entity :-er.add(shortName, longName)
  • Add asset type :-at.add(shortName, longName)
  • Add asset :-ar.add(assetRef, entityShortName, assetTypeShortName, "path")
  • Load principles :-pr.get("path")
  • Setup VQL :-su("path")
  • Rename item :rn(oldName, newName)
  • Delete item :dl(itemName)
